# 小希项目 (hermes-xiaoxi) · 产品方案 v3.0

> 便携AI语音助手 · 开源硬件 + 开源固件 · 2026

---

## 核心理念

**ESP32只管收音播放，Agent后端管所有智能。**

ESP32固件编译一次不用再改，所有设置通过Web页面修改。换Agent后端？改个地址就行。加新功能？后端加，ESP32完全不知道。

## ESP32设备端详细设计

### 固件功能（编译一次，长期使用）

| 模块 | 功能 | 说明 |
|------|------|------|
| 音频采集 | I2S麦克风 → Opus编码 | 16kHz单声道，60ms帧 |
| 音频播放 | Opus解码 → I2S喇叭 | 24kHz输出 |
| 唤醒词 | ESP-SR WakeNet | 桌面版用"你好小鑫"，笔形版用按钮 |
| WiFi管理 | 自动连接 + AP配网 | 家里WiFi / 手机热点自动切换 |
| Web配置 | AP热点模式下的配置页面 | 手机浏览器访问192.168.4.1 |
| 按钮 | 单击/长按/双击 | 消抖，GPIO中断 |
| LED | 状态指示 | 待机/录音/思考/播放 |
| 屏幕 | OLED/LCD显示 | 对话文字、状态、时间 |
| OTA | 固件远程升级 | 通过Web页面上传新固件 |
| 省电 | Light Sleep | 无操作30秒休眠，按钮/唤醒词唤醒 |

### 连接方式

ESP32通过WiFi连接到Agent后端，支持两种场景：

**在家**：
```
ESP32 → 家里WiFi路由器 → 局域网 → Agent后端（192.168.1.11）
```

**在外**：
```
ESP32 → 手机热点 → 互联网 → Agent后端（云端地址）
```

ESP32开机自动搜已知WiFi列表，找不到就等用户手机开热点。用户完全无感。

### 通信协议

ESP32和Agent后端之间用**WebSocket**：

```
① ESP32 → Agent：hello消息（设备信息、音频参数）
② Agent → ESP32：hello应答（会话ID、协商参数）
③ ESP32 → Agent：音频数据（Opus编码，二进制帧）
④ Agent → ESP32：音频数据（TTS结果，Opus编码）
⑤ ESP32 → Agent：控制消息（listen/abort，JSON）
```

## Agent后端详细设计

### 后端是什么

Agent后端就是**ESP32的大脑**。它接收ESP32发来的音频，做完识别→思考→合成后，把回复音频发回去。

后端可以是任何支持小智WebSocket协议的Agent服务：

| 后端 | 说明 | 适用场景 |
|------|------|----------|
| **Hermes（小希）** | 本地电脑运行，功能最强 | 在家日常使用 |
| **兼容Agent平台** | 任何实现相同协议的后端 | 未来扩展 |
| **小智官方服务器** | xiaozhi.me | 最简单，免费Qwen模型 |
| **自建后端** | Docker部署的xiaozhi-server | 技术用户 |
| **其他AI Agent** | 任何实现相同协议的后端 | 扩展性 |

### Hermes后端（小希）

Hermes运行在本地电脑上，是功能最强的后端：

```
ESP32 → WebSocket → Hermes
                      ├── ASR（Whisper / 本地语音识别）
                      ├── 上下文管理（多轮对话记忆）
                      ├── LLM（DeepSeek / 千问 / Claude / 本地模型）
                      ├── TTS（Edge TTS / GPT-SoVITS）
                      ├── 工具调用
                      │   ├── 天气查询
                      │   ├── 搜索引擎
                      │   ├── 智能家居控制
                      │   ├── 日程管理
                      │   └── MCP自定义工具
                      └── Web管理后台
```

### 切换后端

只需要在ESP32的Web配置页面改一个地址：

```
想连Hermes？   → 填 ws://192.168.1.11:8000/xiaozhi/v1/

想连官方服务器？ → 填 ws://api.tenclass.net/xiaozhi/v1/
```

**不需要改固件，不需要重编译，改个地址就切换。**

## 与小智的核心区别

| | 小智（原版） | 小希（我们的方案） |
|---|---|---|
| **后端** | 写死连官方服务器 | 地址可配，随便换 |
| **改设置** | 重编译固件 | Web页面改，秒生效 |
| **Agent能力** | 取决于后端 | 接Hermes=全功能Agent |
| **换模型** | 改固件+重刷 | 后端换，ESP32不知道 |
| **加工具** | 改固件+重刷 | 后端加，ESP32不知道 |
| **离线使用** | 需要自建服务器 | 电脑跑Hermes就行 |
| **配网** | 需要电脑客户端 | ESP32自带Web页面 |

**一句话：小智把ESP32当主角，咱们把ESP32当哑巴终端。**

## 数据流（一次完整对话）

```
① 用户按按钮（或说"你好小鑫"）
      ↓
② ESP32开始录音，LED变绿
      ↓
③ 用户说完，松开按钮（或VAD检测到静音）
      ↓
④ ESP32把Opus音频通过WebSocket发给Agent后端
      ↓
⑤ Agent后端：
   a. ASR识别：音频 → 文字
   b. 上下文管理：拼上前几轮对话
   c. LLM推理：生成回复文字（流式）
   d. 工具调用：如果需要查天气/搜索等
   e. TTS合成：文字 → 音频
      ↓
⑥ Agent后端把Opus音频通过WebSocket发回ESP32
      ↓
⑦ ESP32解码音频，播放到喇叭，LED变蓝
      ↓
⑧ 如果屏幕版：同步显示对话文字
```
