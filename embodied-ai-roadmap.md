# 小希 · 具身智能三步走架构

> 目标：让小希从"能说话的音箱"进化成"能看、能听、能动"的具身智能体
> 硬件基础：ESP32终端 + Y7000本地服务器（4060 8G）
> 模型方向：VLA（Vision-Language-Action）视觉-语言-动作模型

## 三步走路线

### 阶段一：听觉智能（耳朵 + 嘴巴）
> 让小希"听得懂、说得出"，实现自然语音对话

**目标：** ESP32 采集音频 → 服务器 ASR+LLM+TTS → ESP32 播放

**现状：** ✅ 基本已跑通（小智私有后端 Docker 已部署）

**待完善：**
- [ ] 唤醒词优化（"你好小鑫"唤醒精度测试）
- [ ] 对话延迟优化（端到端 < 2秒）
- [ ] 多轮对话上下文保持
- [ ] 情感状态机（根据对话内容切换情感表情）
- [ ] 噪声环境下的拾音质量（AEC回声消除）

**技术栈：**
| 环节 | 方案 | 备注 |
|------|------|------|
| 音频采集 | ESP32 I2S + INMP441 | 16kHz/16bit |
| 音频编码 | Opus (ESP32端) | 60ms帧，低带宽 |
| 传输 | WebSocket | 双向流式 |
| 语音识别 | Whisper / FunASR | 本地或API |
| 大模型 | Qwen3.5-9B / MiMo | LM Studio本地 |
| 语音合成 | GPT-SoVITS | 甜嗲台湾女声 |

### 阶段二：视觉感知（眼睛）
> 让小希"看得见"，理解周围环境

**目标：** ESP32-CAM 摄像头 → 视频流传输 → 服务器视觉推理 → 环境理解

**需要新增：**
- [ ] ESP32-CAM 模块（或升级到带摄像头的开发板）
- [ ] 视频流采集和传输（MJPEG / H.264 over WebSocket）
- [ ] 本地视觉模型部署（VLM）
- [ ] 多模态融合（视觉+语言联合推理）

**技术栈：**
| 环节 | 方案 | 备注 |
|------|------|------|
| 图像采集 | ESP32-CAM / OV2640 | 640×480@10fps |
| 图像编码 | JPEG 压缩 | 平衡质量和带宽 |
| 传输 | WebSocket / HTTP POST | 逐帧或关键帧 |
| 视觉理解 | Qwen2.5-VL-7B / RoboBrain2.0-7B | 4060可跑 |
| 场景描述 | VLM 自然语言描述 | 结合语音交互 |

**视觉能力分层：**
1. **基础视觉** — "我看到了什么"（物体识别、场景描述）
2. **空间理解** — "东西在哪里"（位置关系、距离估计）
3. **动态感知** — "发生了什么"（动作识别、变化检测）
4. **视觉对话** — 用户问"桌上有什么"→ 小希看一眼回答

## VLA 模型方案

### 什么是 VLA？
VLA = Vision-Language-Action，一个模型同时处理：
- **Vision**：看懂摄像头画面
- **Language**：理解人类语言指令
- **Action**：输出机器人动作（关节角度、移动方向等）

### 候选模型

| 模型 | 参数量 | 4060可跑？ | 特点 |
|------|--------|-----------|------|
| OpenVLA-7B | 7B | ✅ 可跑 | 开源VLA，基于Llama2+SigLIP |
| π0 (Physical Intelligence) | 3B | ✅ 可跑 | 轻量VLA，适合桌面机器人 |
| RoboBrain2.0-7B | 7B | ✅ 已下载 | 视觉理解强，需适配动作输出 |
| OpenVLA-Mini | 3B | ✅ 轻松 | OpenVLA蒸馏版 |

### 本地运行路径
```
LM Studio 加载 VLA 模型 (GGUF量化)
     ↓
API 端口 :1234
     ↓
Hermes Agent 调用
     ↓
ESP32 终端执行动作
```

## 数据流时序

```
用户: "小希，帮我看看桌上有什么"

1. ESP32 麦克风 → 音频采集 → Opus编码
2. WebSocket → 发送到服务器
3. 服务器 ASR → "帮我看看桌上有什么"
4. LLM 理解意图 → 需要视觉 → 触发摄像头
5. ESP32-CAM → 拍照 → JPEG → 发送到服务器
6. VLM 视觉推理 → "桌上有：一杯咖啡、一个键盘、一本笔记本"
7. TTS 合成语音 → "桌上有一杯咖啡、一个键盘和一本笔记本哦~"
8. WebSocket → 音频流回 ESP32
9. ESP32 喇叭播放

用户: "把咖啡推到左边去"

10. ASR → "把咖啡推到左边去"
11. VLA 模型：视觉(咖啡位置) + 语言(推到左边) → 动作序列
12. 动作指令 → WebSocket → ESP32
13. ESP32 舵机执行：移动到咖啡位置 → 推 → 回到原位
14. TTS: "好嘞，咖啡已经推到左边了~"
```
